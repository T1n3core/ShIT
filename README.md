# shit — a minimal Git-like clone

A lightweight educational reimplementation of core Git concepts.  
Work in progress.

## Features
- Initialize a repository with a `.shit/` directory  
- SHA-1–named objects stored under `objects/`  
- Basic object creation (blobs, trees, commits)  
- Simple command structure (`shit init`, `shit add`, etc.)  
- Transparent object headers modeled after Git  
- Focus on understanding Git internals rather than full functionality
